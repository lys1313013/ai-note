# Corrective RAG 与 RAG 变体对比

## RAG 流水线分类

```
┌─────────────────────────────────────────────────────┐
│                   RAG 家族                          │
├──────────────┬──────────────┬──────────────┬────────┤
│   Naive RAG  │  Semi-RAG    │ Advanced RAG │Agentic │
│  (基本检索)  │ (带后处理)   │ (含优化)     │(Agent) │
└──────────────┴──────────────┴──────────────┴────────┘
```

## 各变体详解

### 1. Naive RAG（朴素 RAG）
```
Query → Retrieve → Generate → Answer
```
- 最基础的形态，直接拼接检索结果
- 缺点：噪声多、语义漂移、幻觉严重

### 2. Corrective RAG（纠错 RAG）

核心思想：**在生成之前和之后都检查检索/生成质量，有问题则纠正**。

```
Query → Retrieve → [质量评估] → [纠错/补充检索] → Generate → [幻觉检测] → Answer
                         ↓                              ↓
                    检索太烂就重检                  幻觉就重写
```

代表工作：
- **CRAG**（Corrective-RAG）：检索结果太差 → 用 Web Search 兜底
- **REFEED**：生成后检查幻觉，补充检索缺失信息

## CRAG 核心逻辑

```python
def correct_rag(query, retrieved_docs):
    # Step 1: 评估检索质量
    relevance_scores = [score(doc, query) for doc in retrieved_docs]
    avg_score = sum(relevance_scores) / len(relevance_scores)
    
    # Step 2: 根据评分决定后续动作
    if avg_score < threshold:
        # 检索质量差，改用 Web Search 兜底
        web_docs = web_search(query)
        retrieved_docs = web_docs
    elif avg_score < medium_threshold:
        # 中等质量，补充检索
        extra_docs = expand_retrieve(query)
        retrieved_docs = retrieved_docs + extra_docs
    
    # Step 3: 生成 + 幻觉检测
    answer = generate(query, retrieved_docs)
    if is_hallucination(answer, retrieved_docs):
        answer = regenerate(query, retrieved_docs)
    return answer
```

## 各 RAG 变体对比

| 变体 | 检索优化 | 生成优化 | 适用场景 |
|------|---------|---------|----------|
| Naive RAG | 无 | 直接生成 | 简单场景快速验证 |
| Advanced RAG | Query改写/Rerank | 上下文压缩 | 通用生产环境 |
| Corrective RAG | 质量评估+重检 | 幻觉检测+重写 | 高可靠性要求 |
| Self-RAG | 模型自判断按需检索 | 选择性引用 | 资源敏感场景 |
| Agentic RAG | 动态多跳规划 | Agent 决策生成 | 复杂多跳问答 |

## 面试常问问题

**Q: CRAG 和 Self-RAG 的区别？**
CRAG 关注检索和生成结果的外部纠正（质量评估+兜底），Self-RAG 是模型内部的自我判断机制（反思token）。

**Q: 什么时候用 Agentic RAG，什么时候用普通 RAG？**
简单单跳问题直接用普通 RAG（成本低、延迟低）。复杂多跳、需要多工具配合、开放式问题用 Agentic RAG。

**Q: CRAG 的纠错机制怎么设计？**
核心是两个阈值：检索质量阈值（决定是否重检）、幻觉检测阈值（决定是否重写）。阈值需要根据业务场景调优。
