# LangGraph 实现 Agentic RAG

## 为什么用 LangGraph？

LangGraph 是 LangChain 生态的图执行框架，用于构建有状态、多步骤的 LLM 应用。

**核心概念**：
- **State**：跨节点共享的数据
- **Node**：一个处理函数（如 RAG 检索、LLM 生成）
- **Edge**：节点之间的连接 + 条件跳转

## Agentic RAG 的 LangGraph 实现

```python
from langgraph.graph import StateGraph, END

class RAGState(TypedDict):
    question: str
    retrieved_docs: List[Document]
    answer: str
    iterations: int

def retrieve_node(state: RAGState):
    """检索节点"""
    docs = vector_search(state["question"])
    return {"retrieved_docs": docs}

def generate_node(state: RAGState):
    """生成节点"""
    answer = llm.generate(state["question"], state["retrieved_docs"])
    return {"answer": answer}

def should_continue(state: RAGState):
    """判断是否继续检索"""
    if state["iterations"] >= 3:
        return "end"
    # 评估检索结果质量
    if is_retrieval_quality_insufficient(state["retrieved_docs"]):
        return "retrieve"  # 继续检索
    return "end"

# 构建图
graph = StateGraph(RAGState)
graph.add_node("retrieve", retrieve_node)
graph.add_node("generate", generate_node)
graph.set_entry_point("retrieve")
graph.add_conditional_edges("retrieve", should_continue)
graph.add_edge("generate", END)

app = graph.compile()
```

def route_after_retrieve(state):

## 面试常问问题

**Q: LangGraph 和 LangChain 的区别？**
LangChain 是工具库，LangGraph 是状态机。LangGraph 用于构建复杂的多步骤、有循环的 LLM 应用（就像 Agentic RAG），而 LangChain 更偏向单步工具调用。

**Q: Agentic RAG 中状态管理怎么做？**
用 TypedDict 定义 State，每个节点返回要更新的字段。LangGraph 自动 merge 更新到全局状态。复杂场景可以用 Redis 做分布式状态管理。
