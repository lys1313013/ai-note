# 知识图谱与 RAG 面试汇总

## 核心对比表

| 问题 | 知识图谱 | RAG |
|------|---------|-----|
| 知识表示 | 图结构（实体+关系） | 非结构化文本块 |
| 推理能力 | **强**（多跳遍历） | 弱（依赖文本匹配） |
| 可解释性 | **高**（关系路径） | 中（文本来源） |
| 构建成本 | 高 | 低 |
| 适用问答 | 关系推理、事实查询 | 摘要、开放问答 |

## 面试必答题

**Q: 什么时候用知识图谱而非 RAG？**
需要多跳关系推理（如"A公司的CTO和B公司的CEO的关系"）、需要精确事实查询、知识结构本身是关系型的（人物/公司/产品网络）。

**Q: Graph RAG 的构建成本怎么控制？**
先用 LLM 抽取高频实体/关系（频率优先），小规模验证效果再扩展；实体消歧用 Embedding 相似度做粗筛+人工抽检。

**Q: TransE 的核心思想？**
头实体 + 关系 ≈ 尾实体。如：马云 + 创立 ≈ 阿里巴巴。把关系建模为向量空间的"翻译"。

**Q: 实体消歧怎么做？**
上下文 Embedding 与知识库实体匹配 + 协同消歧（利用图结构关系辅助判断）+ LLM 兜底。

## Graph RAG 记忆口诀

```
Graph RAG = 知识图谱 + RAG
构建：文本 → 三元组抽取 → 图谱
检索：子图扩展 / 路径查询 / 社区检测
优势：多跳推理强、可解释性高
缺点：构建成本高、实体消歧难
```
