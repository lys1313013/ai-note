# 向量检索与 Embedding

## 什么是 Embedding？

Embedding（嵌入）是将文本、图像、音频等数据转换为固定维度向量的技术。

**核心目标**：相似的文本在向量空间中距离更近。

```
"苹果水果"  → [0.12, -0.34, 0.78, ...]
"苹果手机"  → [0.11, -0.31, 0.75, ...]
"橙子"      → [0.89, 0.21, -0.15, ...]
```

水果"苹果"和"橙子"距离近，手机"苹果"和"三星"距离近——这就是语义相似性。

## Embedding 质量指标

**MTEB (Massive Text Embedding Benchmark)**：业界最权威评测榜单，涵盖 58 个数据集、8 个任务类型。

评估维度：
- **Retrieval**：检索任务（最核心）
- **Clustering**：聚类
- **Classification**：分类
- **Pairwise**：相似度排序

## 相似度度量

| 方法 | 公式 | 适用场景 |
|------|------|----------|
| 余弦相似度 | cos(θ) | 文本相似度（最常用） |
| 点积 | A·B | 归一化后的向量 |
| 欧氏距离 | \|A-B\| | 距离敏感场景 |

```python
# 余弦相似度
cos_sim = dot(a, b) / (norm(a) * norm(b))
```

## 面试常问问题

**Q: Embedding 模型如何选型？**
- 中文任务 → BGE、M3E（国产）
- 英文任务 → OpenAI、Cohere
- 私有化部署 → BGE、M3E
- 成本优先 → text-embedding-3-small

**Q: 向量维度越高越好？**
不一定。维度越高存储成本越高、检索越慢。高维度向量可能存在"维度灾难"，需要权衡。

**Q: 如何评估 Embedding 质量？**
- MTEB 榜单排名
- 下游任务准确率
- 召回率/精确率
