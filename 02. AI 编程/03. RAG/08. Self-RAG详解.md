# Self-RAG 详解

## 什么是 Self-RAG？

Self-RAG（Self-Retrieval Augmented Generation）是斯坦福 2024 年提出的工作，核心思想：**让 LLM 学会自我检索、自我判断回答质量。**

> 普通 RAG：检索 → 拼接 → 生成（不管检索质量）
> Self-RAG：生成前先问自己"需要检索吗？"，生成后再问"回答是否基于检索内容？"

## 两种反思token

### 1. Retrieve Token（决定是否检索）

模型在生成过程中自行判断是否需要外部检索。

```
思考：这个问题需要检索吗？
输出：Retrieve: Yes/No
```

### 2. IsGround Token（判断是否基于检索）

模型生成每个陈述后，判断是否有检索证据支持。

```
生成：Transformer是一种注意力机制...
输出：IsGround: [YES]  # 基于检索内容
生成：Attention is all you need...
输出：IsGround: [NO]   # 模型自身知识
```

## Self-RAG 训练方法

1. **生成反思样本**：用强模型（如 GPT-4）生成带有反思标签的数据
2. **训练轻量模型**：在标注数据上微调 LLM（7B/8B 量级）
3. **在线推理**：生成时并行输出反思 token，动态控制检索

关键训练 loss：
```
L = L_generation + λ * L_retrieval + μ * L_groundedness
```

## 与普通 RAG 对比

| 维度 | 普通 RAG | Self-RAG |
|------|---------|----------|
| 检索触发 | 固定（100%触发） | 按需（模型自判断） |
| 检索结果处理 | 全量拼接 | 选择性使用 |
| 幻觉检测 | 无 | 有（IsGround） |
| 计算成本 | 固定 | 按需，更省 |

## 面试常问问题

**Q: Self-RAG 的反思token是怎么训练出来的？**
用强模型（GPT-4）生成大量带反思标签的数据，然后在这些数据上微调一个7B/8B的轻量模型，让它学会自判断。

**Q: Self-RAG 的缺点？**
训练成本高，需要大量标注数据。模型可能过度依赖自身判断而漏掉重要检索。
