# 知识图谱嵌入 (KG Embedding) 与图神经网络

## 为什么要做 KG Embedding？

知识图谱是符号结构（实体/关系是符号），LLM 是向量空间。需要把 KG 投影到向量空间才能和 LLM 结合。

**核心任务**：
- 实体嵌入：每个实体 → 向量
- 关系嵌入：每种关系 → 向量/矩阵

## 主流方法

### 1. TransE（翻译嵌入）

**核心思想**：头实体 + 关系 ≈ 尾实体

```
h + r ≈ t
vec(马云) + vec(创立) ≈ vec(阿里巴巴)
```

简单高效，是 KG Embedding 的 baseline。

### 2. TransR / TransD（多空间翻译）

TransE 所有实体共享同一空间，实际关系可能需要多语义空间。

```
TransR：头/尾实体在实体空间，关系在独立空间投影
TransD：引入实体和关系的动态投影矩阵
```

### 3. RotatE（大模型友好）

把关系建模为复数空间的旋转：
```
h ○ r = t  （○ 表示旋转操作）
```
可建模：对称性（相同实体）、反对称性（不同类型关系）、逆关系。

### 4. 神经网络方法

| 方法 | 说明 |
|------|------|
| **GCN/GAT** | 图卷积/注意力聚合邻居信息 |
| **BERT-GCN** | BERT 获取实体文本表示 → GCN 编码结构 |
| **KG-BERT** | 把三元组当句子做 BERT 分类 |

## KG Embedding + RAG 应用

```python
# 实体向量入库
entity_embeddings = entity_encoder(entities)
vector_db.upsert(entity_embeddings)

# 检索时
query_emb = encode("马云创立的公司")
# 既检索文本块，也检索图谱实体
results = vector_db.search(query_emb, top_k=10)

# 生成时附上检索到的关系路径
context = "关系路径：马云 --创立--> 阿里巴巴\n来源：知识图谱"
```

## 面试高频问题

**Q: TransE 的缺点？**
无法建模一对多、多对一、多对多关系（如"执导"关系一部电影有多位导演）。

**Q: 如何选择 KG Embedding 方法？**
小规模图谱 + 精度优先 → RotatE；大规模图谱 + 推理效率 → TransE；需要捕捉复杂关系语义 → GAT。
