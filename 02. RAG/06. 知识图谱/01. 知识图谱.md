# 知识图谱

知识图谱（Knowledge Graph，KG）是一种以**实体-关系-实体**三元组为基本单位，描述客观世界事物及其关联的语义网络。

## 核心概念

- **实体（Entity）** — 客观存在的事物，如"北京"、"爱因斯坦"
- **关系（Relation）** — 实体间的关联，如"位于"、"出生于"
- **属性（Attribute）** — 实体的特征，如"人口"、"出生日期"
- **三元组（Triple）** — `(头实体, 关系, 尾实体)`，知识图谱的最小单位
- **本体（Ontology）** — 知识图谱的 Schema，定义类别、属性、关系类型

## 例子

```
(北京, 是...的首都, 中国)
(北京, 人口, 2185万)
(爱因斯坦, 出生于, 德国)
(爱因斯坦, 获得, 诺贝尔物理学奖)
```

## 知识图谱 vs 关系数据库

| 维度 | 关系数据库 | 知识图谱 |
|------|------------|----------|
| 模型 | 表格（行/列） | 图（节点/边） |
| 关系 | 外键 JOIN | 边（可带属性） |
| 灵活性 | Schema 固定 | 灵活可扩展 |
| 推理 | 弱 | 支持图推理 |
| 适用 | 强结构数据 | 关系密集、复杂 |

## 构建流程

```
数据源（文本/数据库/网页）
    ↓
信息抽取（命名实体识别、关系抽取）
    ↓
知识融合（实体对齐、消歧）
    ↓
知识加工（推理、补全）
    ↓
知识存储（图数据库）
```

## 主流工具

- **Neo4j** — 最流行的图数据库，Cypher 查询语言
- **JanusGraph** — 分布式图数据库
- **NebulaGraph** — 国产高性能图数据库
- **Amazon Neptune** — AWS 托管
- **TigerGraph** — 企业级
