# 知识图谱核心概念

## 什么是知识图谱？

用图结构表示知识的语义网络：**实体（节点）+ 关系（边）**。

```
[哈利波特] ---(主演)--→ [丹尼尔·雷德克里夫]
[哈利波特] ---(类型)--→ [奇幻]
[丹尼尔·雷德克里夫] ---(出生)--→ [1989年]
```

本质：结构化的知识库，支持高效的多跳关系推理。

## 知识图谱三要素

| 要素 | 说明 | 示例 |
|------|------|------|
| **实体 (Entity)** | 现实世界的事物 | 人名、公司、产品 |
| **关系 (Relation)** | 实体间的连接 | 主演、工作、位于 |
| **属性 (Attribute)** | 实体的特征 | 出生日期、国籍 |

## 知识图谱 vs RAG

| 维度 | 知识图谱 | RAG |
|------|---------|-----|
| 知识表示 | 图结构 | 非结构化文本 |
| 推理能力 | 强（多跳查询） | 弱 |
| 构建成本 | 高（需抽取） | 低（直接切分） |
| 可解释性 | 高（关系链清晰） | 中 |
| 适用场景 | 关系查询、推理 | 问答、摘要 |

## 核心操作：知识抽取

从非结构化文本中抽取出三元组。

**方法**：
- **NER**（命名实体识别）→ 实体
- **RE**（关系抽取）→ 关系
- **EE**（事件抽取）→ 更复杂的事件

**常用模型**：
- BERT+CRF（NEP/RE）
- GCN/Transformer（关系编码）
- LLM Prompt（few-shot 抽取）

## 面试高频问题

**Q: 知识图谱的构建流程？**
文本 → 分句 → NER → 实体标准化 → RE抽取关系 → 知识融合（对齐消歧）→ 图谱存储

**Q: 实体消歧怎么做？**
相似度匹配（Embedding）、上下文消歧、协同消歧（利用全局知识）。

**Q: 知识图谱的存储？**
Neo4j（原生图）、RDF store（学术）、图数据库（NebulaGraph/Milvus）。企业常用多模存储（KG+向量库）。
