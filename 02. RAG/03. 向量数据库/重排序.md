# 重排序（Reranker）

Reranker 是 RAG 检索链路的第二阶段，用于**对初检结果精排**，提升 Top-K 准确率。

## 为什么需要重排序

向量检索（Bi-Encoder）速度快但精度有限——它独立编码 query 和 doc，没有交互。Reranker（Cross-Encoder）会**联合编码 query+doc**，精度更高但慢。

```
向量检索 (Top-100) → Reranker 精排 → Top-5 → LLM
```

## 启动 Reranker 服务

使用 HuggingFace Text Embeddings Inference（TEI）：

```bash
docker run -itd \
  --name text-embeddings-inference \
  --gpus all \
  -p 8080:80 \
  -v /home/data:/data \
  ghcr.io/huggingface/text-embeddings-inference:1.6 \
  --model-id BAAI/bge-reranker-large
```

## 主流 Rerank 模型

| 模型 | 特点 |
|------|------|
| **BAAI/bge-reranker-large** | 中文场景 SOTA |
| **BAAI/bge-reranker-v2-m3** | 多语言 |
| **Cohere Rerank 3** | 商业级 API |
| **Jina Reranker** | 商业级 API |

## Python 使用

```python
from sentence_transformers import CrossEncoder

model = CrossEncoder("BAAI/bge-reranker-large")
scores = model.predict([
    ("query", "doc1"),
    ("query", "doc2"),
])
```

## LangChain 集成

```python
from langchain.retrievers import ContextualCompressionRetriever
from langchain.retrievers.document_compressors import CrossEncoderReranker

compressor = CrossEncoderReranker(model=CrossEncoder("BAAI/bge-reranker-large"))
compression_retriever = ContextualCompressionRetriever(
    base_compressor=compressor,
    base_retriever=vector_retriever,
)
```
