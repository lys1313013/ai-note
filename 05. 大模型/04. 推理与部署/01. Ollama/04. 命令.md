# 命令


```
ollama help
```


启动

```
ollama serve
```


查看模型信息

```
ollama show deepseek-r1:7b
```


在命令行运行（最后会显示token数量）
```bash
# ollama run <模型名称> --verbose
ollama run qwen3:32b --verbose
```


返回结果示例：
```
total duration:       25.060584236s               
load duration:        21.534815ms        
prompt eval count:    20 token(s)        
prompt eval duration: 69.391383ms           
prompt eval rate:     288.22 tokens/s       
eval count:           675 token(s)       
eval duration:        24.969064399s     
eval rate:            27.03 tokens/s
```

|                      |                                                  | 说明                                                         |
| -------------------- | ------------------------------------------------ | ------------------------------------------------------------ |
| total duration       | 整个推理过程（从加载模型到生成完整输出）的总耗时 | 包括模型加载、提示词处理和文本生成的全部时间                 |
| load duration        | 模型加载到内存/显存所需的时间                    | 此过程仅需一次（后续调用无需重复加载），与模型大小和硬件性能相关 |
| prompt eval count    | 提示词（Prompt）被分割成的 Token 数量            | Token 是模型处理的最小文本单位（如单词或子词）。例如，输入 `"你好，世界"` 可能被拆分为 `["你", "好", "，", "世界"]`（4 Tokens） |
| prompt eval duration | 处理提示词（Prompt）所需的时间                   | 模型对输入的 20 Tokens 进行编码和理解的过程耗时              |
| prompt eval rate     | 提示词处理速度（Token 吞吐率）                   | `prompt eval count / prompt eval duration` = `20 / 0.06939 ≈ 288.22` **说明**：此阶段速度较快，因为提示词可并行处理（Transformer 架构优势） |
| eval count           | 生成文本（Response）的 Token 数量                | 模型实际生成的文本长度（如回答内容）                         |
| eval duration        | 生成文本（Response）所需的时间                   |                                                              |
| eval rate            | 文本生成速度（Token 吞吐率）                     |                                                              |

time ollama run deepseek-r1:32b "测试"
