## 关键工程问题

### 1. 索引构建（Indexing Pipeline）

```python
# 文档处理流水线
def process_documents(docs):
    for doc in docs:
        # 1. 解析（PDF/Word/Markdown）
        raw_text = parse(doc)
        # 2. 分块（chunking）
        chunks = chunk(raw_text, strategy="semantic")
        # 3. Embedding
        embeddings = embed_model.encode(chunks)
        # 4. 元数据提取（标题、来源、日期）
        metadata = extract_metadata(doc)
        # 5. 写入向量库
        vector_db.upsert(chunks, embeddings, metadata)
```

### 2. 向量库选型

| 场景 | 推荐 | 理由 |
|------|------|------|
| 初创/快速验证 | Chroma | 轻量、Python 原生 |
| 中等规模生产 | Qdrant | 高性能、Cloud Native |
| 大规模企业 | Milvus/Pinecone | 分布式、高可用 |
| PG 已有技术栈 | PGVector | 复用现有数据库 |

### 3. 缓存策略

```python
# Query 结果缓存
@cache(ttl=3600)  # 1小时缓存
def cached_retrieve(query, top_k):
    return vector_search(query, top_k)

# Embedding 缓存（相同文本不重复 embedding）
@lru_cache(maxsize=10000)
def cached_embed(text):
    return embed_model.encode(text)
```

### 4. 监控与可观测性

关键指标：
- 检索召回率（Recall）
- 生成答案质量（ faithfulness）
- P99 延迟
- Token 消耗成本

```python
# 埋点监控
def rag_pipeline(query):
    start = time.time()
    docs = vector_search(query)
    answer = generate(query, docs)
    latency = time.time() - start
    
    metrics.log({
        "query": query,
        "num_docs": len(docs),
        "latency_ms": latency * 1000,
        "token_used": estimate_tokens(answer)
    })
    return answer
```

## 面试常问问题

**Q: RAG 生产环境最大的挑战是什么？**
端到端质量保证 + 延迟成本平衡。检索和生成都会引入延迟，需要分级降级策略（简单问题用小模型/少检索，复杂问题再加大投入）。

**Q: 如何保证 RAG 系统的稳定性？**
多级兜底（向量检索→关键词检索→Web搜索→直接回答），监控召回率和幻觉率，设置人工兜底接口。

**Q: 怎么做 RAG 的 A/B 测试？**
核心指标：答案质量（Faithfulness/Relevancy）、用户满意度、响应延迟。分组对照不同召回策略、Embedding 模型、Prompt 模板。
