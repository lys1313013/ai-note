# Graph RAG 原理与实践

## Graph RAG 是什么？

把知识图谱引入 RAG 流程：**用图谱组织知识 → 图谱检索 → LLM 生成**。

传统 RAG：切分文本块 → 向量检索 → 拼 prompt
Graph RAG：构建知识图谱 → 图检索 → 关系链 → 生成

## 核心优势：多跳关系推理

```python
# 问：马云和阿里巴巴是什么关系？
# 图谱检索路径：
(马云) --(创立)--> (阿里巴巴)  # 直接关系
(马云) --(担任董事长)--> (阿里巴巴)  # 另一路径

# 传统 RAG：很难把两个实体串起来
# Graph RAG：沿着关系边遍历，自然得到推理链
```

## 构建方法

**LLM 抽取三元组**：
```python
prompt = """
从文本中提取知识三元组（实体1, 关系, 实体2）：
文本：苹果公司由乔布斯创立，总部位于加州库比蒂诺。
三元组：
(苹果公司, 创始人, 乔布斯)
(苹果公司, 总部位于, 加州库比蒂诺)
"""
```

**工具**：KgBuilder、OpenIE、spaCy + GAT

## 检索策略

| 策略 | 说明 |
|------|------|
| **子图检索** | 扩展邻居实体，捕获上下文 |
| **路径检索** | 找两实体间的路径 |
| **社区检测** | 实体聚类，优先检索同类实体 |

## 微软 GraphRAG 核心思想

1. 从文档抽取实体 → 构建局部图
2. 用 Leiden 算法做社区检测（社区内高度互联）
3. 每个社区生成摘要（Community Summary）
4. **两层检索**：先查 Community Summary 定位话题 → 再查具体实体

**优点**：大幅压缩 token（文档→社区→摘要），适合大规模知识库。

## 面试高频问题

**Q: Graph RAG 和传统 RAG 的核心区别？**
传统 RAG 检索文本块，Graph RAG 检索实体和关系链。Graph RAG 更擅长关系推理，多跳问答效果更好。

**Q: Graph RAG 的缺点？**
构建成本高（需要 LLM 抽取图谱），实体消歧难，小规模知识库优势不明显。
