# RAG 检索优化

## 基础检索：向量相似度检索

```python
# 典型流程
query_embedding = embed_model.encode("用户问题")
results = vector_db.search(query_embedding, top_k=5)
```

## 问题：简单向量检索的局限

- **语义漂移**：用户问题用词和文档不一致
- **同义词问题**：用户说"电脑"，文档写"计算机"
- **Query太长/太短**：Embedding 效果差

## 检索优化方法

### 1. Query 改写（Query Rewriting）

用 LLM 将用户问题改写成更适合检索的表述。

```python
# 原始 Query: "那个关于机器学习的课程讲的啥"
# 改写后: "机器学习课程内容介绍"
```

工具：
- RewriteRM（华为诺亚）
- Query2Doc（微软）
- HyDE（假设性文档）

### 2. HyDE（假设性文档检索）

让 LLM 生成一个"假设性答案"，用这个答案去检索。

```
Query → LLM生成假设答案 → 用假设答案检索
```

### 3. 混合检索（Hybrid Search）

结合关键词检索（BM25/TF-IDF）和向量检索，两者取并集或加权融合。

```python
hybrid_score = alpha * vector_similarity + (1-alpha) * bm25_score
```

### 4. 重排序（Reranker）

初排用向量检索拉回 Top-K，Reranker 进一步排序。

常用模型：
- BGE-Reranker（国产最强）
- Cohere Rerank
- Cross-Encoder

### 5. 元数据过滤

检索时加上过滤条件（时间、类别、来源），减少候选集。

```python
results = vector_db.search(
    query_embedding,
    filter={"category": "技术文档", "year": 2024},
    top_k=10
)
```

### 6. 上下文压缩（Context Compression）

将检索到的块用 LLM 压缩，提取最相关信息，减少 token 消耗。

## 多跳检索（Multi-Hop）

复杂问题需要多步检索：

```
Q: "A公司的产品在B公司有哪些应用？"
→ Step 1: 检索A公司产品列表
→ Step 2: 基于Step1结果检索B公司应用
→ 综合回答
```

框架：IRCoT（Chain of Thought）、ReAct

## 面试常问问题

**Q: 如何提升 RAG 召回率？**
- 优化 Embedding 模型（换用 BGE 等中文优化模型）
- 增加 Query 改写
- 混合检索
- 调整分块策略

**Q: Reranker 的作用？**
向量检索在高维空间近似搜索可能有精度损失，Reranker 用 Cross-Encoder 做精确排序，提升最终结果质量。

**Q: 检索和生成怎么权衡？**
检索是找到正确的内容（Precision/Recall），生成是基于内容回答（Fluency）。两者都重要，但检索是生成质量的前提。
