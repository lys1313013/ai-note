# 多跳检索（Multi-Hop Retrieval）

## 什么是多跳？

单跳（Single-Hop）：一次检索直接得到答案。
多跳（Multi-Hop）：需要多次检索 + 推理才能回答。

**典型例子**：
- Q: "A 公司 CEO 和 B 公司 CTO 是什么关系？" → 需先分别查两人身份
- Q: "比特币在 2020 年的涨幅是多少？" → 需先知道 2020 年价格再算涨幅

## 多跳的挑战

| 问题 | 说明 |
|------|------|
| 中间答案丢失 | 各跳结果需要传递给下一跳 |
| 错误级联 | 跳1错 → 跳2错 → 最终错 |
| 检索盲目 | 不知道跳1会返回什么，难以设计跳2 |
| 循环检索 | 可能陷入重复检索死循环 |

## 主流方法

### 1. IRCoT（Chain of Thoughts for Retrieval）

在检索过程中加入思维链，让每跳的推理可见。

```
Q: A公司和B公司的关系？
Step 1: 检索 A 公司 CEO → 李明
Step 2: 基于"李明"检索"李明 B公司" → 发现李明是B公司独立董事
Answer: 李明同时担任A公司CEO和B公司独立董事
```

### 2. ReAct（Reasoning + Acting）

让 Agent 在"推理"和"行动"之间交替：
- Reason：思考目前知道什么，需要什么
- Act：执行一个行动（检索/工具调用）
- Observe：观察行动结果

### 3. Subquery Planning（子查询规划）

在回答前，将复杂问题拆解为多个子问题。

```python
# 输入：问题拆解
subqueries = [
    "A公司CEO是谁",
    "B公司CTO是谁",
    "这两人之间的关系"
]
# 分别检索，再综合
```

## 面试常问问题

**Q: 多跳检索最大的难点？**
错误级联 + 中间结果管理。每一跳的错误会放大到下一跳，需要有评估和回退机制。

**Q: 如何避免多跳检索的死循环？**
限制最大跳数（通常 3-4 跳），用 visited set 记录已检索的主题避免重复。

**Q: IRCoT 和 ReAct 的区别？**
IRCoT 专注于检索策略的链式推理；ReAct 更通用，结合推理和工具执行。IRCoT 是 ReAct 在 RAG 场景的特化。
