# 实体消歧与知识融合

## 实体消歧（Entity Disambiguation）

**问题**：同一文本在不同上下文指代不同实体。

```
"苹果" → 水果苹果？Apple公司？
"乔丹" → 篮球之神？经济学家？
```

**方法**：
1. **上下文 Embedding 匹配**：将文本中的"苹果"和知识库中的"苹果公司"编码后比较相似度
2. **协同消歧**：利用知识图中其他实体关系辅助判断
3. **LLM 消歧**：Few-shot prompt 让 LLM 根据上下文选择正确实体

```python
# 示例
prompt = """
文本："库克带领苹果公司创新"
知识库实体：
- 苹果公司（美国科技公司，总部加州）
- 苹果（水果）
判断：文本中的"苹果"指哪个实体？
答案：苹果公司
"""
```

## 知识融合（Knowledge Fusion）

**问题**：不同来源的知识可能有冲突或重复。

```
来源A：[张三，出生于，北京]
来源B：[张三，出生于，上海]
→ 需要冲突检测和消解
```

**融合层次**：
- **实体对齐**：A的"张三"和B的"张某人"是同一人
- **关系合并**：同一关系不同描述，取置信度高的
- **冲突消解**：多来源冲突时按优先级/时间戳选择

## 知识图谱构建完整流程

```
原始数据（文本/表格/数据库）
    ↓
知识抽取（NER → RE → EE）
    ↓
实体标准化（消歧、去重、归一化）
    ↓
知识对齐（跨源实体对齐）
    ↓
关系冲突消解
    ↓
图谱存储（Neo4j/Milvus） + 向量化
    ↓
上层应用（RAG、推理、问答）
```

## 面试高频问题

**Q: 实体消歧的难点？**
歧义性（同名实体）、上下文稀缺、新实体识别（知识库没有的实体）。

**Q: 如何评估知识融合质量？**
准确率（对齐正确的比例）、召回率（漏掉的比例）、F1 分数。人工抽检是 gold standard。
